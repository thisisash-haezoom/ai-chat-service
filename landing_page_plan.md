# 🚀 AI TEAM MATE - Landing Page PRD & UI/UX Plan

## 1. 개요 및 기획 의도
*   **서비스명:** AI TEAM MATE
*   **목표:** 방문자가 기존 AI 툴의 한계(귀찮은 반복 학습, 단발성 대화)를 공감하고, 다중 AI 협업이라는 완벽한 솔루션에 매료되어 즉각적으로 `[ 시작하기(로그인) ]` 버튼을 누르도록 기획.
*   **디자인 컨셉 (VS Code Aesthetic):** 기존 앱의 'VS Code / IDE' 감성을 200% 살려, 다크 톤 배경과 터미널 폰트(Monospace), 코드 신택스 하이라이팅을 세일즈 카피에 적용합니다. 매끄러운 스크롤 애니메이션(Framer Motion)을 더해 "개발자스럽고 힙한" 타겟팅 랜딩 페이지를 구축합니다.

---

## 2. 페이지 섹션 구성 (Flow)

### Section 1: Hero (대표 섹션 - 첫 인상)
*   **헤드라인:** "코딩부터 디자인, 법률 검토까지. 당신의 비즈니스를 위한 완벽한 전속 AI 팀."
*   **서브카피:** "명령어 한 줄이면 충분합니다. 1인 기업도 이제 대기업처럼 일하세요."
*   **비주얼 컨셉:** 터미널 창 화면 중앙에 타이핑 애니메이션으로 `> recruiting... UX Designer, Backend Dev, Legal Advisor... [Done]` 메시지가 전환됨.
*   **강렬한 CTA:** `[ 터미널 접속 및 팀 구성하기 ]` 버튼 (클릭 시 `/login` 리다이렉트).

### Section 2: The Problem (우리가 겪는 고통 - 공감 자극)
*   **헤드라인:** "매번 새로운 창을 열고, 배경 상황을 앵무새처럼 설명하는 일에 지치셨나요?"
*   **상세 페인포인트 (Error Console 시각화):**
    *   `Error 404:` GPT/Gemini에게 수십 번 맥락(Context)을 학습시켜도 결국 엉뚱한 대답이 돌아오는 불편함.
    *   `Error 500:` 전문적인 프롬프트 지식이 없어 AI의 성능의 20%밖에 쓰지 못하는 현실.
    *   `Fatal Error:` 한 명의 비서로는 부족합니다. 병렬적으로 여러 직군의 작업 프로세스를 연결할 수 없음. 

### Section 3: The Target (누구를 위한 서비스인가?)
*   **헤드라인:** "혼자서 거대한 성과를 만들어야 하는 당신을 위해."
*   **타겟 고객군 (코드의 Interface 정의 형식으로 표현):**
    *   `{ role: "CEO", need: "내 아이디어를 실행해줄 실무진. 1인 기업/스타트업 대표님들" }`
    *   `{ role: "Office Worker", need: "내 전문 분야 밖의 지식(법무/회계/번역)이 급히 필요한 직장인" }`
    *   `{ role: "Team Leader", need: "반복적인 실무를 든든하게 받쳐줄 서브 팀원이 필요한 실무자" }`

### Section 4: The Solution & Core Features (우리의 해결책)
*   **헤드라인:** "검증된 Workflow가 탑재된 전문가들을 내 Workspace에 초대하세요."
*   **주요 기능 시각화 (IDE 패널 UI 흉내내기):**
    1.  **플러그 앤 플레이:** 필요한 직군의 AI 팀원을 마켓플레이스에서 버튼 한 번으로 추가/삭제(Install/Uninstall).
    2.  **독립된 영구 채널 (Persistent Context):** 휘발되지 않는 채팅 채널. 한 번 설정한 프로젝트의 맥락은 영원히 유지됩니다.
    3.  **칸반 대시보드 (Highlight!):** 대표님이 채팅으로 지시를 내리면, 여러 AI 에이전트(기획자, 디자이너, 개발자)가 병렬로 작업하는 현황을 칸반(Kanban) 보드 UI로 시각화합니다. 대표님은 보고 탭에서 진행률을 확인하고 `[Approve (컨펌)]`만 하시면 됩니다.

### Section 5: Pricing (비용)
*   **헤드라인:** "당신의 전속 기획/개발팀을 유지하는 가장 합리적인 방법."
*   **티어 구성 (JSON 포맷을 활용한 디자인):**
    *   `[ Free ]` : 개인 토이 프로젝트용 (기본 에이전트 2명 제한, 기본 모델, 공용 컨텍스트)
    *   `[ Basic ]` : 1인 창업가/프리랜서용 (에이전트 무제한 마켓 접근, 채널 생성 무제한, 프리미엄 Workflow 사용 가능)
    *   `[ VIP ]` : 기업형 풀스택 팀 구축용 (칸반 대시보드 무제한, 최고성능 AI 모델 우선 접근권, 우리 회사 데이터 커스텀 학습 허용)

### Section 6: Final CTA (바텀 액션)
*   **헤드라인:** "`> initialize_ai_team.sh`"
*   **메시지:** "이제 지시만 내리세요. 실무는 저희가 합니다."
*   **버튼:** 웅장하고 눈에 띄는 `[ 지금 바로 나의 AI 팀 시작하기 ]` -> `/login` 라우팅.

## 3. 제안 및 다음 단계 (Next Steps)
현재 앱의 최상위 경로(`/`)가 채팅 페이지로 설정되어 있습니다. 
완벽한 제품 구조를 위해 다음과 같은 아키텍처 변경을 제안합니다.

1. **라우팅 분리:** 
   - `page.tsx` (`/`) -> **위 기획이 담긴 메인 랜딩 페이지**로 교체
   - 사용자가 로그인하면 -> `/chat` (또는 `/dashboard`, 현재 메인 채팅 레이아웃)로 리다이렉트되도록 내부 로직 재정비.
2. **랜딩 페이지 UI 컴포넌트 개발:**
   - Framer Motion을 사용하여 스크롤 시 각 섹션이 타이핑되거나 페이드인되는 애니메이션 집중 구현.
   - VS Code 컨셉에 어울리는 `CodeBlock`, `TerminalCard`, `PricingBox` 컴포넌트 퍼블리싱.

위 기획안 수정사항이나 승인(컨펌)을 명령해주시면, 즉시 **라우팅 분리 및 랜딩 페이지 완벽 구현** 코드 작업을 시작하겠습니다!
